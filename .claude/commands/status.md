# /status — 项目进度概览

> 快速了解项目当前状态、进度和待处理事项。

---

## 执行步骤

### Step 1：收集数据

读取以下文件并统计：

1. **docs/TODO.md**：
   - 总 TODO 数
   - 已完成数（`- [x]`）
   - 未完成数（`- [ ]`）
   - 下一个待执行 TODO

2. **docs/STATE.md**：
   - 当前阶段 / Phase
   - 最近更新时间
   - 当前活跃 TODO
   - Blocker（如有）

3. **docs/FEEDBACK.md**：
   - Open 条目数
   - 各 open 条目的简要标题

4. **bug_fix/**：
   - 未解决的 bug 文件数
   - 各 bug 文件名

5. **docs/PRD.md**：
   - 审批状态

### Step 2：输出概览

```
═══════════════════════════════════════
  项目状态概览
═══════════════════════════════════════

  进度:  ████████░░░░░░░░  12/25 (48%)

  当前阶段:   Phase 2 — {阶段描述}
  活跃 TODO:  T112 — {描述}
  最近更新:   2026-03-26

───────────────────────────────────────
  待处理事项
───────────────────────────────────────

  Bug（紧急）:   2 个
    - fix-login-error.md
    - fix-api-timeout.md

  Feedback（约束）: 1 个 open
    - F003: API 返回格式统一

  PRD 状态:  approved

───────────────────────────────────────
  下一步建议
───────────────────────────────────────

  → 有未修复 Bug，建议先运行 /execute 处理
  → 或运行 /execute 继续执行 T112
```

### Step 3：建议下一步

根据状态给出建议：

| 状态 | 建议 |
|------|------|
| 有 bug_fix/ 未处理 | → 运行 `/execute`，Bug 会被优先处理 |
| 有未完成 TODO | → 运行 `/execute` 继续执行 |
| TODO 全部完成 | → 项目已完成，或添加新需求 `/brainstorm` |
| PRD 为 draft | → 审核 PRD 并改为 approved，然后 `/plan` |
| 无 TODO 无 PRD | → 运行 `/brainstorm` 开始新功能规划 |

---

## 规则

- 只读不写，不修改任何文件
- 进度条使用 Unicode block 字符可视化
- 如果文件不存在或为空，对应区域显示"（未初始化）"
- 输出简洁，一屏可看完
